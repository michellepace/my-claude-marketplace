# Get Next.js Docs

The `.nextjs-docs/` directory and its `INDEX.md` contain official Next.js documentation as a reference for Claude Code — not humans. Trim the documentation to what is relevant for the current project so Claude does not waste context on irrelevant material.

Follow these steps in order. Steps 1-3 run without confirmation. Step 4 requires user confirmation before continuing. Use emojis for readability.

## Step 1: Clean slate

Remove any existing docs to ensure a fresh state. This handles both first-time setup and re-runs (e.g. after a Next.js upgrade). `.next-docs/` is the raw docs tree generated by the codemod in Step 2; `.nextjs-docs/` is the consolidated directory this skill creates.

```bash
rm -rf .nextjs-docs .next-docs
```

## Step 2: Generate docs

```bash
npx @next/codemod@latest agents-md --help # Understand tool
mkdir -p .nextjs-docs && npx @next/codemod@latest agents-md --output .nextjs-docs/INDEX.md
```

The codemod creates two things:
- `.nextjs-docs/INDEX.md` — the compact index consumed by Claude Code
- `.next-docs/` — the full documentation tree (at the project root)

## Step 3: Consolidate

Move the docs into `.nextjs-docs/` so everything is self-contained, then fix the root path in the index:

```bash
mv .next-docs .nextjs-docs/
if grep -q 'root: ./.next-docs' .nextjs-docs/INDEX.md; then
  sed -i.bak 's|root: ./.next-docs|root: ./.nextjs-docs/.next-docs|' .nextjs-docs/INDEX.md \
    && rm -f .nextjs-docs/INDEX.md.bak \
    && echo '✅ Success: INDEX.md root path updated'
else
  echo '❌ Error: Could not find "root: ./.next-docs" in INDEX.md — Read the file, find the root: segment, and Edit the path to ./.nextjs-docs/.next-docs'
fi
```

If a `.markdownlint-cli2.yaml` file exists at the project root, ensure `.nextjs-docs/` is listed under `ignores`.

## Step 4: Suggest directories to remove

Inspect the documentation tree:

```bash
tree -L 3 -a .nextjs-docs
```

Then inspect the project to understand how it uses Next.js — check the project structure (`app/` or `pages/`), `next.config.*`, `package.json`, etc.

Review each top-level directory inside `.nextjs-docs/.next-docs/` and identify any that are irrelevant as a Claude Code reference for this project. Suggest only top-level directories to remove with reasoning. Example:

## Step 5: Remove confirmed directories and update index

Remove the confirmed directories from disk, then run the strip script to remove their entries from the index. The script automatically verifies that the remaining index entries match the files on disk.

```bash
rm -rf .nextjs-docs/.next-docs/<dir1> .nextjs-docs/.next-docs/<dir2> \
  && uv run ${CLAUDE_SKILL_DIR}/scripts/strip-nextjs-index.py .nextjs-docs/INDEX.md \
    --remove <dir1> --remove <dir2>
```

Replace `<dir1>`, `<dir2>`, etc. with the actual directory names (e.g. `02-pages`, `04-community`).
