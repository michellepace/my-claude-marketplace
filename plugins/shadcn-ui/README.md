# Plugin: `shadcn-ui`

Best practices for building UIs with shadcn/ui components (Radix UI + Tailwind CSS). Covers component customisation, CVA variants, composition patterns, and recent changes beyond Claude's training data.

## What's Inside

## Usage Examples

- `/shadcn-ui` I need a "soft" button variant with a muted background from my theme
- `/shadcn-ui` How should I structure a settings page with sidebar nav and form sections?
- `/shadcn-ui` Replace the deprecated Toast component — what's the current approach?
